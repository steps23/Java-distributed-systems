Assegnamento 2 Sistemi Distribuiti - Stefano Ruggiero mat.3649891. PanoramicaQuesto progetto implementa un sistema distribuito con:  • Barrier iniziale (REGISTER / START via multicast UDP)    • Mutual exclusion centralizzata basata su coda FIFO (REQUEST / GRANT / RELEASE via TCP)    • Failure detection tramite heartbeat UDP    • Bully Election Algorithm per eleggere un nuovo coordinatore in caso di crash    • Simulazione crash del coordinatore con probabilità configurabile  Contiene 5 classi Java nel package `stefano_ruggiero_assegnamento_2`:  - Config.java  - Message.java  - UDPMessenger.java  - Coordinator.java  - Node.javaTutte le altre spiegazioni dettagliati sono presenti nel javadoc presente nella cartella.2. CompilazioneAprire un terminale nella cartella `assegnamento_2` e digitare :  javac -d out stefano_ruggiero_assegnamento_2/*.java3. EsecuzionePrima di lanciare i processi assicurarsi di non aver già nulla in ascolto sulle porte: UDP 4446 e TCP 4444.a) Avviare il Coordinatore: in un terminale alli’interno della cartella “assegnamento_2” avviare il comando “ java -cp out stefano_ruggiero_assegnamento_2.Coordinator ”; b) Avviare i Nodi: In altri terminali (uno per nodo), fornire un ID univoco da 1 a N (N = EXPECTED_NODES = 3 nel file config al momento) e avviare i comandi “ java -cp out stefano_ruggiero_assegnamento_2.Node 1 ”, “ java -cp out stefano_ruggiero_assegnamento_2.Node 2 ” , “ java -cp out stefano_ruggiero_assegnamento_2.Node 3 ” c) Logica di esecuzione:    • Il Coordinatore attende le registrazioni, invia START e poi heartbeat periodici     • I Nodi si registrano, entrano nel ciclo REQUEST?GRANT?RELEASE fino a RESOURCE_TARGET     • Se il Coordinatore “crasha” (simulato), un Nodo lancerà Bully Election e diventerà nuovo Coordinatore     • Alla fine, Coordinatore invia DONE e tutti i processi terminano  4. Parametri di configurazioneSi possono modificare i seguenti parametri nel file Confing.java:  • Numero di nodi (`EXPECTED_NODES`)    • Target di risorse per nodo (`RESOURCE_TARGET`)    • Intervallo e timeout di heartbeat    • Probabilità di crash (`CRASH_PROBABILITY`)  Per verificare le modifiche, salvare il file e ricompilare.
