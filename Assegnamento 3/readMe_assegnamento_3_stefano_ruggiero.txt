PanoramicaQuesto progetto implementa lıalgoritmo di snapshot distribuito di Chandy-Lamport su un insieme di nodi interconnessi via TCP. Caratteristiche principali:  Ñ Generazione casuale di ÊTASKË tra i nodi    Ñ Cattura periodica e casuale (10ð30 s) dello stato globale tramite messaggi MARKER    Ñ Registrazione del contatore locale e del buffer dei canali in ingresso    Ñ Terminazione automatica dopo un numero configurabile di snapshot o su comando ÊquitË    Ñ Shutdown pulito con chiusura di socket, thread pool e scheduler  Contiene 5 classi Java nel package `stefano_ruggiero_assegnamento_3`:  - `Message.java`    - `SnapshotState.java`    - `Node.java`    - `NodeMain.java`  Tutte le altre spiegazioni dettagliate sono presenti nel Javadoc generato nella cartella `docs`.2. CompilazioneAprire un terminale nella cartella di progetto e digitare:javac -d out stefano_ruggiero_assegnamento_3/*.java3. EsecuzionePrima di lanciare i processi assicurarsi che le porte TCP a partire da `basePort` (es. 4500) siano libere.a) Avviare ciascun nodo in un terminale separato fornendo:   - `id`           : ID univoco del nodo [0 ╔ Nð1]     - `nodes`        : numero totale dei nodi (N=5 di default)     - `basePort`     : porta base per il ServerSocket (es. 4500)     - `[targetSnap]` : numero massimo di snapshot prima dello shutdown (opzionale)     Esempio per 5 nodi, basePort=4500 e terminazione dopo 3 snapshot:java -cp out stefano_ruggiero_assegnamento_3.NodeMain 0 5 4500 3java -cp out stefano_ruggiero_assegnamento_3.NodeMain 1 5 4500 3╔java -cp out stefano_ruggiero_assegnamento_3.NodeMain 4 5 4500 3b) Shutdown manuale     Ñ Digitare ÊquitË nella console di un nodo per far partire lo shutdown pulito     Ñ Oppure inviare `Ctrl-C` (gestito da hook JVM)
